# PLAN - Phase별 개발 계획

각 Phase는 항상 "실제로 실행 가능한 SW"를 목표로 한다. 고객님(사용자)께서 Phase가 끝날 때마다 직접 실행해보고 피드백을 주시면, 그 피드백을 반영해 다음 Phase로 넘어가는 방식으로 진행한다.

---

## Phase 1. 메인 화면

**목표**
- `npm run dev`로 실행했을 때 게임 타이틀과 시작 버튼이 보이는 첫 화면 구현
- `docs/FEATURES/main.md` 기준의 최소 구성(타이틀, 시작 버튼, 조작 안내)

**고객님이 확인하실 부분**
- 화면에 "PANG" 타이틀과 시작 버튼이 정상적으로 보이는지
- 시작 버튼(또는 스페이스바)을 눌렀을 때 다음 화면(게임 화면)으로 넘어가는지
- 화면 크기를 조절해도 레이아웃이 깨지지 않는지

---

## Phase 2. 플레이어 이동 & 작살 발사

**목표**
- 화면 하단에 플레이어 캐릭터 등장
- 좌우 방향키로 이동, 스페이스바(또는 클릭)로 작살(와이어) 발사
- 아직 풍선은 없는 상태로, 순수 조작감만 확인하는 단계

**고객님이 확인하실 부분**
- 캐릭터가 좌우로 부드럽게 움직이는지, 화면 밖으로 나가지 않는지
- 작살을 쐈을 때 화면 위쪽까지 올라갔다가 자연스럽게 사라지는지
- 연속으로 여러 번 발사했을 때 이상하게 겹치거나 끊기지 않는지

---

## Phase 3. 풍선 등장 및 튕기는 물리

**목표**
- 화면에 큰 풍선이 등장해 중력 영향을 받으며 벽/바닥에 튕기며 이동
- 아직 작살과의 충돌 처리는 없는 단계 (풍선이 자연스럽게 움직이는지만 확인)

**고객님이 확인하실 부분**
- 풍선이 통통 튀는 움직임이 자연스러운지 (너무 빠르거나 느리지 않은지)
- 풍선이 화면 밖으로 빠져나가지 않는지
- 여러 개가 동시에 있을 때 서로 이상하게 겹치거나 멈추지 않는지

---

## Phase 4. 풍선 분열 & 충돌(HP) 시스템

**목표**
- 작살이 풍선에 맞으면 `docs/FEATURES/game_rule.md` 기준으로 4단계 크기로 분열/소멸
- 플레이어가 풍선에 닿으면 HP가 1씩 감소 (초기 HP 10)

**고객님이 확인하실 부분**
- 큰 풍선을 맞췄을 때 두 개의 작은 풍선으로 잘 나뉘는지
- 가장 작은 풍선을 맞췄을 때 완전히 사라지는지
- 풍선에 캐릭터가 닿았을 때 HP가 화면에 표시되고 실제로 줄어드는지
- HP가 0이 되었을 때 어떤 일이 일어나는지 (다음 Phase에서 목숨 처리와 연결됨)

---

## Phase 5. 목숨(Life) 시스템 & Mission 1 클리어/게임오버

**목표**
- `docs/FEATURES/mission1.md` 기준으로 목숨 3개, HP 소진 시 목숨 차감 후 재시작
- 모든 목숨 소진 시 게임 오버 화면
- 화면 내 모든 풍선을 제거하면 Mission 1 클리어 화면
- 이 시점에서 Mission 1이 처음부터 끝까지 플레이 가능한 완성된 게임이 됨

**고객님이 확인하실 부분**
- HP가 0이 되었을 때 목숨이 1개 줄고, HP가 다시 10으로 채워져 스테이지가 재시작되는지
- 목숨을 모두 잃었을 때 게임 오버 화면이 뜨고, 재시작할 수 있는지
- 모든 풍선을 제거했을 때 클리어 화면이 뜨는지
- 전체적으로 처음(메인 화면)부터 끝(클리어/게임오버)까지 막힘 없이 플레이가 가능한지

---
